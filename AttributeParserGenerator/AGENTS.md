# AttributeParserGenerator

Roslyn source generator that helps **other** source generators turn `AttributeData` into strongly typed objects efficiently.

## Scope

- **Primary solution:** `AttributeParserGenerator.slnx` (this directory)
- **Sibling shared code (read-mostly):** `..\SourceGeneratorTools\SourceGeneratorTools.Shared\`
  - Imported via `SourceGeneratorTools.projitems` into `AttributeParser.SourceGenerator`
  - Provides `SourceBuilder` and `ComparableArray<T>`
- This monorepo has other solutions (`SourceGeneratorTools`, `ParamsSourceGenerator`, etc.). Prefer not to change them unless the shared API must extend.

## SDK / TFMs

- SDK from `global.json`: **10.0.203** (`rollForward: latestFeature`)
- Test runner: Microsoft.Testing.Platform
- Analyzer + library: **netstandard2.0** (`AttributeParser`, `AttributeParser.SourceGenerator`)
- Samples / tests / benches / test infra: **net10.0**

## How it works

1. Consumer marks a `static partial` method with `[AttributeParser]`.
2. Method must take exactly `AttributeData` and `AttributeDataParser` (any order).
3. Return type **properties** define the output shape (`ImmutableArray<T>` → `GetValues<T>()`, else `GetValue<T>()`).
4. Generator emits a partial that:
   - initializes locals
   - `foreach` over `parser.Parse(attributeData)`
   - `switch` on argument name (PascalCase + camelCase)
   - returns via **property object initializer** (`new T() { ... }`)

Pipeline files:

- `AttributeParserGenerator.cs` — `Initialize` / grouping
- `AttributeParserGenerator.CollectData.cs` — symbols → records / `Error`
- `AttributeParserGenerator.GenerateSource.cs` — emit via `SourceBuilder`
- `Data/*` — equality-safe pipeline records (`ComparableArray` for collections)

Consumer example (`AttributeParser.TestInfrastructure/AttributeParsers.cs`):

```csharp
[AttributeParser]
public static partial InputWithDifferentTypes ParseInputWithDifferentTypes(
    AttributeData attributeData,
    AttributeDataParser parser);
```

## Shared tooling

### `SourceBuilder` (`SourceGeneratorTools`)

Indent-aware source emitter. Prefer over raw `StringBuilder`.

- `AddAutoGeneratedComment()`, `#nullable enable`
- `Append` / `AppendLine` (including interpolated handlers)
- `CreateBlock()` / `CreateLine()` — use `using` scopes for braces and multi-part lines
- Do **not** use `Environment.NewLine` in generators

### `ComparableArray<T>`

Value-equality `IReadOnlyList<T>` for incremental generator caching / record equality. Use on any collection that participates in pipeline equality (e.g. `ParserClassData.ParserMethods`).

## Conventions

- Keep CollectData pure (symbols → records); emit only in GenerateSource.
- Split generator/runtime into partials by phase / nested type.
- Diagnostics: `APG####`, category `AttributeParser.SourceGenerator` (today: `APG1000`).
- Output model fields are **properties**, not fields.
- SampleCode must **not** reference the generator; only TestInfrastructure does.
- Analyzer/library stay netstandard2.0 and analyzer-safe (`EnforceExtendedAnalyzerRules`).

## Future work (priority)

1. **Constructor initialization** of return types (only property init is implemented).
2. **Custom field parsers** for advanced per-field scenarios.
3. **Broader tests** — generator diagnostics, invalid signatures, emitted-source coverage (runtime parser tests exist; generated-path coverage is thin).

When implementing features, prefer the backlog above over drive-by refactors. Do not break existing property-init consumers.
